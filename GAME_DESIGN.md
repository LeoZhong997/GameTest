# 黑棘行垒 — 游戏设计文档

> 一款以反派视角展开的军团自动战斗养成游戏

## 一、游戏基本信息

- **游戏名称**：黑棘行垒（Legion Rush）
- **游戏类型**：自动战斗 + 阵容构筑 + 轻量养成
- **核心体验**：反派统帅、移动要塞、自动战斗、阵容策略
- **游戏引擎**：Cocos Creator 3.8.8（2D）
- **开发语言**：TypeScript
- **目标平台**：微信小游戏 / iOS / Android

### 一句话定位

玩家率领一座移动战争要塞，沿旧大陆推进，招募各方危险兵种，战前布阵，战中看技能自动连锁爆发，以轻决策体验获得持续爽感。

### 产品气质

- 阴暗但不压抑
- 反派但不纯恶
- 战争感强，操作负担低
- 有构筑乐趣，没有卡牌操作疲劳

## 二、世界观

### 2.1 背景设定

大陆"艾嵴"曾被"王冠同盟"统一统治。多年战争后，同盟开始以"秩序"为名，大规模清洗边境领地、异族部族和不受控的研究组织。

玩家曾是同盟麾下的边境统帅，因拒绝执行"焦土清剿令"而被扣上"叛逆者"与"灾厄总督"的罪名。军团被剿灭，名誉被抹除，只剩一座半毁的移动要塞 **黑棘行垒**。

如今率领残兵与流亡者重新出发，在世人眼中是反派、叛徒、掠夺者；但也在撕开旧秩序的谎言。

### 2.2 玩家身份

**黑棘行垒之主 / 逆征总督**

- 被通缉的旧帝国统帅
- 反派视角的军事统治者
- 有决断，不是疯子
- 目标：活下去、集结力量、复仇并重塑边境秩序

### 2.3 移动要塞

黑棘行垒是一座能沿古代轨道行进的战争堡垒，是玩家主城的叙事来源：

| 要塞区域 | 对应功能 |
|---------|---------|
| 兵营 | 招募与整编兵种 |
| 熔炉 | 强化装备、重铸遗器 |
| 档案室 | 解锁敌人资料、兵种背景 |
| 议事台 | 主线推进、阶段决策 |
| 监牢/审讯室 | 收编俘虏、触发事件 |
| 征募台 | 抽卡/征募兵种 |
| 遗器库 | 圣物/遗器管理 |
| 军需库 | 背包/物品管理 |

## 三、核心玩法

### 3.1 战斗系统

- **规模**：双方各最多 5 种兵种 × 数量由等级决定
- **操控**：战前部署阵容 + 选择军令 → 战斗自动进行
- **战斗时长**：最长 180 秒
- **核心循环**：推图 → 获取资源 → 升级兵种 → 推进更高关卡

### 3.2 轻决策原则

一场战斗中，玩家只需完成：

1. **选上阵兵种**（最多 5 种）
2. **调整站位**（3×3 九宫格）
3. **选择军令**（可选，1 条战前 buff）
4. **开战后观看战斗反馈**

### 3.3 自动技能

所有兵种技能在战斗中 **自动释放**，触发方式：

- 普攻次数触发
- 能量条满触发
- 生命值低于阈值触发
- 入场若干秒后触发
- 击杀后触发

### 3.4 军令系统

每场战斗可选 **1 条军令**（战前选一次，复用 buff 逻辑实现）：

| 军令 | 效果 |
|------|------|
| 强袭令 | 开场前 8 秒全军攻速 +20% |
| 固守令 | 前排获得护盾，等效生命 +30% 持续 6 秒 |
| 追猎令 | 全军速度 +20%，持续整场 |
| 抑咒令 | 敌方后排攻击 -30%，持续 3 秒 |

## 四、阵营与兵种

### 4.1 四大阵营

代码层 ID 不变（`human/beast/spirit/demon`），显示层为原创阵营名。

## 五、养成与经济

### 5.1 货币体系

| 货币 | 代码ID | 显示名 | 获取 | 用途 |
|------|--------|--------|------|------|
| 主货币 | gold | 铁券 | 推图、离线收益 | 升级、抽卡 |
| 高级货币 | crystals | 黑徽 | 首通、Boss关 | 圣物抽卡、商店 |
| 圣物强化 | relic_essence | 遗烬 | 遗器副本、分解 | 升级遗器 |

### 5.2 征募系统（抽卡）

- 兵种征募：消耗铁券，单抽/十连，十连保底紫色
- 遗器征募：消耗黑徽，品质概率 绿60%/蓝28%/紫10%/金2%，十连保底蓝

### 5.3 兵种养成

- **升级**：消耗经验书（战训手册）提升等级，属性按成长值增加
- **升阶**：消耗灵魂碎片 + 升阶令，提升品质
- **遗器装备**：每兵种 1 个遗器槽位，提供百分比属性加成

### 5.4 遗器系统

- 14 件遗器：4 阵营专属 + 5 角色专属 + 5 通用
- 主属性（随机类型）+ 副属性（每 5 级增加 1 个，最多 3 个）
- 最高 20 级，消耗铁券 + 遗烬升级

## 七、剧情方向

### 7.1 核心叙事

主线不是"打败魔王拯救世界"，而是：

**玩家作为被定义为反派的人，一边活下去，一边拆穿真正制造灾难的秩序。**

### 7.2 前三章

| 章 | 标题 | 关键事件 |
|----|------|---------|
| 1 | 黑棘启程 | 处刑场脱身 → 重新点亮行垒 → 收拢旧部 → 与追兵交锋 |
| 2 | 荒盟之血 | 进入荒原 → 接触裂骨荒盟 → "收编 or 威压"抉择 → 旧帝国压迫边地 |
| 3 | 灰塔通缉 | 王冠同盟正式通缉 → 俘虏/密报/内鬼事件 → 牺牲型决策 |

### 7.3 核心角色

| 角色 | 身份 | 功能 |
|------|------|------|
| 维尔莎 | 行垒副官 | 冷静务实，原帝国参谋，主城界面叙事 |
| 霍恩 | 荒盟使者 | 粗野但讲信用，荒盟支线 |
| 赛弥亚 | 暮林观测者 | 神秘克制，秘法支线 |
| 厄罗肯 | 烬术囚徒 | 危险偏执，深烬支线 |

### 7.4 叙事风格

- 角色说话简洁
- 多写立场冲突、交易、背叛、招安、审判
- 让玩家感到在经营一支危险但真实的军团

## 十、技术方案

### 10.1 架构模式

- **组件化**：Entity（兵种单位）+ Component（技能、AI、属性）+ System（战斗、渲染）
- **事件驱动**：全局 EventBus 解耦各系统
- **数据驱动**：所有数值配置外置 JSON
- **状态机**：单位用 FSM 管理（Idle/Move/Attack/Skill/Dead）
- **对象池**：频繁创建销毁的单位需复用

```typescript
export const SCENE_LABELS = { gameTitle: '黑棘行垒', units: '兵  营', ... };
```

代码层 ID（`human/beast/spirit/demon`）不变，只改显示名。
